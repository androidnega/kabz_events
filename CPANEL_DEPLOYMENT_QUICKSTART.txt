STEP 8: INSTALL COMPOSER & DEPENDENCIES (10-15 minutes)
========================================================

8.1 Check if Composer is installed:
    composer --version

8.2 If NOT installed, install Composer:
    cd ~
    curl -sS https://getcomposer.org/installer | php
    mkdir -p ~/bin
    mv composer.phar ~/bin/composer
    chmod +x ~/bin/composer
    echo 'export PATH="$HOME/bin:$PATH"' >> ~/.bashrc
    source ~/.bashrc

8.3 Verify Composer is working:
    composer --version

    Should show: Composer version 2.x.x

8.4 Navigate back to project:
    cd ~/laravel/kabzs-event

8.5 Install PHP dependencies:
    composer install --optimize-autoloader --no-dev

    ‚è≥ This will take 5-15 minutes depending on server speed
    
    If you get memory errors, use:
    php -d memory_limit=-1 ~/bin/composer install --optimize-autoloader --no-dev

‚úÖ CHECKPOINT: Vendor folder is created and filled with packages

STEP 9: CONFIGURE ENVIRONMENT & DATABASE (10 minutes)
======================================================

9.1 Create .env file from example:

    (Then paste the template from ENV_CONFIGURATION_TEMPLATE.txt)

9.2 Edit .env file:
    nano .env

    Use arrow keys to navigate, edit these lines:

    APP_ENV=production                  ‚Üê Change from "local"
    APP_DEBUG=false                     ‚Üê Change from "true"
    APP_URL=https://yourdomain.com      ‚Üê Your actual domain

9.5 Seed database with initial data:

    ‚è≥ This creates admin accounts, categories, settings, etc.

==========================================================

================================================================================
                        TROUBLESHOOTING
================================================================================

PROBLEM: "500 Internal Server Error"
SOLUTION:
  chmod -R 775 storage bootstrap/cache
  php artisan cache:clear
  Check logs: tail storage/logs/laravel.log

PROBLEM: "Database connection failed"
SOLUTION:
  - Verify credentials in .env match exactly from Step 5
  - Ensure DB_HOST=localhost (not 127.0.0.1)
  - Test: php artisan tinker ‚Üí DB::connection()->getPdo();

PROBLEM: "404 Not Found" on all pages
SOLUTION:
  - Check .htaccess exists in public folder
  - Contact hosting to enable mod_rewrite

PROBLEM: CSS/Images not loading
SOLUTION:
  - Verify APP_URL in .env matches your domain
  - Check symlink: ls -la ~/public_html
  - Clear browser cache

PROBLEM: Permission denied errors
SOLUTION:
  chmod -R 775 storage bootstrap/cache
  chown -R username:username ~/laravel/kabzs-event

For more issues, see: CPANEL_GITHUB_DEPLOYMENT_GUIDE.txt (Part 13)
