MOST IMPORTANT CONFIG FILES:
----------------------------

1. config/database.php
   Purpose: Database connection settings
   What it does: Reads DB_HOST, DB_DATABASE, DB_USERNAME, DB_PASSWORD from .env
   When to edit: Usually never - set values in .env instead

2. config/mail.php
   Purpose: Email/SMTP configuration
   What it does: Reads MAIL_HOST, MAIL_PORT, MAIL_USERNAME, MAIL_PASSWORD from .env
   When to edit: Usually never - set values in .env instead

3. config/services.php
   Purpose: Third-party service credentials
   What it does: Reads Paystack, Cloudinary settings from .env
   When to edit: Only if adding new services

4. config/app.php
   Purpose: Application settings (timezone, locale, etc.)
   What it does: Reads APP_NAME, APP_ENV, APP_DEBUG, APP_URL from .env
   When to edit: Rarely - mostly for advanced customization

HOW CONFIG FILES WORK:
----------------------
Config files use env() function to read from .env:

Example from config/database.php:
'host' => env('DB_HOST', '127.0.0.1'),
         ‚Üë                    ‚Üë
    Reads from .env      Default value

So when you set DB_HOST=localhost in .env,
the config file automatically uses it.

================================================================================
                    IMPORTANT NOTES
================================================================================

‚úÖ ALWAYS EDIT .ENV FIRST
   - 99% of configuration changes go in .env
   - Never hardcode passwords in config files
   - Use env() function in config files to read from .env

‚ö†Ô∏è SECURITY WARNINGS
   - Never commit .env to GitHub (it's in .gitignore)
   - Never share .env publicly
   - Always use different .env for local vs production
   - Keep backups of .env in secure location

üîß AFTER EDITING CONFIG
   - Always run: php artisan config:clear
   - In production, run: php artisan config:cache
   - Restart queue workers if using queues
   - Clear browser cache if changes don't appear

üìù CONFIG FORMAT RULES
   - No spaces around = sign: APP_NAME=value ‚úì
   - Use quotes for values with spaces: APP_NAME="My App" ‚úì
   - No trailing spaces: DB_HOST=localhost ‚úì
   - Comments start with #: # This is a comment
   - Blank values are OK: DB_PASSWORD=

================================================================================
                    FREQUENTLY ASKED QUESTIONS
================================================================================

Q: I can't find the .env file!
A: It's hidden. Enable "Show hidden files" in Windows Explorer, or create it
   from .env.example. See "Option 3" under "HOW TO FIND/CREATE IT" above.

Q: Which file has my database password?
A: .env file, look for DB_PASSWORD=

Q: Can I edit config/database.php instead of .env?
A: You can, but it's NOT recommended. Always use .env for credentials.

Q: Do I need to restart anything after editing .env?
A: Run: php artisan config:clear (and config:cache in production)

Q: What's the difference between .env and .env.example?
A: .env.example is a template with no real values. Copy it to .env and fill
   in your actual passwords/credentials. .env.example goes to GitHub, .env doesn't.

Q: My changes to .env aren't working!
A: Clear the config cache: php artisan config:clear

Q: Where do I put my Paystack API key?
A: In .env file: PAYSTACK_SECRET_KEY=sk_test_xxxxx

Q: Should I edit .env or config files on the server?
A: Edit .env on the server at ~/laravel/kabzs-event/.env

Q: Can I have different configs for local and production?
A: Yes! Use different .env files:
   - Local: C:\xampp\htdocs\kabz\.env (APP_ENV=local, APP_DEBUG=true)
   - Production: ~/laravel/kabzs-event/.env (APP_ENV=production, APP_DEBUG=false)
