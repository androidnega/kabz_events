====================================================================================================
📋 BUSINESS LOGIC OVERVIEW
====================================================================================================

USER TYPES:
1. CLIENT - Browse vendors, leave reviews, bookmark favorites
2. VENDOR - Register business, manage services, respond to inquiries
3. ADMIN - Verify vendors, moderate content, manage featured ads

CORE WORKFLOW:
1. Vendor registers account → becomes 'vendor' role
2. Vendor fills business profile → creates Vendor record
3. Vendor adds services → creates Service records
4. Admin verifies vendor → is_verified = true
5. Clients browse verified vendors
6. Clients contact vendors, leave reviews
7. Vendors can pay for featured placement
8. Vendors can subscribe to premium plans

DATABASE RELATIONSHIPS:
- User hasOne Vendor
- Vendor hasMany Services
- Vendor belongsToMany Categories (through Services)
- Vendor hasMany Reviews
- User hasMany Reviews
- User hasMany Bookmarks
- Service belongsTo Vendor
- Service belongsTo Category
- Review belongsTo Vendor
- Review belongsTo User
